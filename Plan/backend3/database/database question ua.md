# План бази даних застосунку

> Переклад з оригіналу `None ai/working/backend3/database/database questoin.md`. Структура і сенс збережені, граматику виправлено. **Це ідеї, не фінальний план** — для авторитетної специфікації див. `docs/database.md` та `None ai/working/backend3/database/database implementation plan.md`.

## Ідея

Я вважаю, що база даних має бути локальною і знаходитися в `app/db/`. Тут мають бути:

- Сама база даних
- Допоміжні скрипти
- README з поясненням кожної таблиці, що вона містить, її рядків та колонок

База даних має зберігати наступну інформацію:

- Інформація про сесії: id, назва, час початку, час завершення, тривалість, час компактування та інша корисна інформація
- Файли конфігурації

## Що ще може зберігати база даних

Це відкрите питання, яке потребує обговорення. Див. `database implementation plan.md` для повного переліку того, що має зберігатися (templates, instructions, user_style, library_file, session_files, custom_template_annotations, 3 кеш-таблиці, app_settings, secrets, pipeline_runs, audit_log).
