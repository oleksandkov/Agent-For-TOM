# TOM — Idea / Концепція проєкту

## Що таке TOM

**TOM** (Agent-For-TOM) — це локальний AI-агент, що працює на машині користувача. Це desktop-додаток із сесіями (як ChatGPT/Claude/Gemini desktop), який вміє підключатися до будь-якої LLM — локальної (Ollama) або хмарної (OpenAI, Anthropic, Google, будь-який OpenAI-сумісний endpoint).

## Проблема

Існуючі AI-асистенти мають дві ключові проблеми:

1. **Stateless** — кожна нова сесія починається з нуля. Жодного контексту про юзера, його проєкти, робочий стиль.
2. **Vendor lock-in** — прив'язка до одного провайдера, складно перемикати між локальним LLM і хмарою.

Додатково: більшість агентів не мають нормального механізму **самонавчання** — не здатні виявляти повторювані патерни й адаптуватися.

## Рішення

TOM — це:

- **Локальний** — дані на машині юзера, не йдуть у хмару без opt-in.
- **Multi-provider** — будь-яка LLM за вибором юзера.
- **Самовдосконалюваний** — "kill feature": аналізує взаємодії, виявляє патерни, формує структуроване розуміння юзера, може авто-створювати нові скіли (після підтвердження).
- **Розширюваний** — через MCP (Model Context Protocol), власні скіли, плагіни.
- **Самодостатній** — один інсталятор, без залежностей від Python/Node на машині юзера.

## Цільова аудиторія

- Розробники, які хочуть локального AI-асистента без щомісячної плати за хмару
- Дослідники / студенти, що працюють із чутливими даними
- Продуктові люди, які ведуть багато довгострокових проєктів і хочуть, щоб агент "пам'ятав" контекст
- Ентузіасти self-hosted / open-source

## Ключові фічі (MVP)

1. **Desktop-додаток** із сесіями (Tauri-based, чат-перший UX)
2. **Multi-provider LLM** (Ollama, OpenAI, Anthropic, Google, custom endpoint)
3. **MCP-підтримка** — клієнт і сервер
4. **Вбудовані скіли** для DOCX, PDF, форматування
5. **Трирівнева пам'ять** (core / archival / recall) на основі Letta
6. **Self-improvement pipeline** — детекція патернів → генерація скіла → схвалення юзером → активація
7. **Локальна БД** (SQLite + SQLCipher, шифрування через OS keyring)
8. **Sandbox для авто-скілів** (no-network, timeout, read-only fs)
9. **Повний офлайн** при використанні локального LLM (жодної телеметрії, авто-update, CDN)

## Диференціатор: Self-improving memory

Центральна "kill feature" TOM. На відміну від звичайного "збереження історії чату":

- **Core memory** — структуроване розуміння юзера (його ролі, проєкти, преференції). Юзер може редагувати вручну.
- **Archival** — векторне сховище фактів і контексту з минулих сесій.
- **Recall** — recency-буфер для поточного контексту.
- **Pattern detector** — щогодини кластеризує сесії, знаходить повторювані workflow.
- **Skill scaffolder** — коли патерн виявлено ≥N разів, LLM-агент генерує MCP-скіл для автоматизації.
- **Approval queue** — юзер завжди підтверджує перед активацією. Без сюрпризів.

Це наближає TOM до **"AI-колеги, який вивчає вашу роботу"** замість "інструмента з кнопкою чату".

## Брендинг

- **Назва**: TOM (тимчасова робоча назва — можна перейменувати)
- **Ліцензія**: MIT (як у репозиторії)
- **Тон**: технічний, дружній, без пафосу
- **UI**: темна тема за замовчуванням, акцентний колір (TBD), іконки власного дизайну
- **Позиціонування**: "Personal AI agent that learns you"

## Архітектурна основа

TOM побудований як **глибокий форк Langflow** із вбудованим **Letta SDK** (пам'ять) і окремим **Tauri-десктопом** (чат-перший UI). Деталі — у плані реалізації (`tom-implementation.md`).

## Метрики успіху (для MVP)

- Холодний старт інсталятора → перше повідомлення: **< 5 секунд**
- 100% функціональність офлайн з локальним LLM
- Бекап/відновлення стану через копіювання одного каталогу
- Кількість сесій до першого "корисного" pattern detection: **≤ 10**
- Час на схвалення auto-скіла юзером: **< 30 секунд** (preview + одна кнопка)

## Out of scope (idea-level)

- Мультиюзер / авторизація
- Хмарна синхронізація між пристроями
- Мобільний клієнт
- Голосовий інтерфейс
- Тренування / файнтюн моделей
- Соціальні фічі (шеринг сесій, публічні скіли)

Це — лише ідея/концепція. Технічна реалізація, фази, команда, тестування — у `tom-implementation.md`.
