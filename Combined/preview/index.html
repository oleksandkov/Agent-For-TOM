Відкрити
→ Pipeline Flow

Двопрохідна архітектура: Text LLM → Image Processing Stage. 5 вхідних елементів → валідний DOCX + PDF.

STEP 0
Конвертація файлів
PDF/DOCX → текст, дедуплікація по SHA-256, стиснення якщо > 4000 токенів
STEP 1
Text LLM (Pass 1)
5 input elements → filled.py. Prefix-cache для повторюваних запусків
STEP 2
Валідація
ast.parse + перевірка плейсхолдерів + word count. HARD FAIL → STOP
STEP 5
Compose
Вставка PNG у DOCX/PDF, заміна anchor-маркерів
STEP 3
Image Gen
Matplotlib (diagram) + HuggingFace (illustration). Cache по render_spec
STEP 4
Execute
subprocess.run filled.py (AST-validated, sandboxed) → DOCX + PDF
