# Створення власного шаблону (Combined v2)

Користувач може створити власний шаблон з PDF/DOCX файлу — без написання коду.

---

## Повний UI Flow

### Крок 1 — Завантажити файл

Користувач завантажує `.pdf` або `.docx`.

**Відразу після завантаження:**
- Визначити тип по MIME (не по розширенню)
- Конвертувати в текст зі збереженням метаданих форматування (шрифт, розмір, bold/italic, вирівнювання, відступи)
- Показати конвертований текст на панелі анотацій (має з'явитись за 2-3 секунди для файлів <5MB)
- Зберегти оригінал у `library_file` (дедуплікація по SHA-256: якщо файл вже є — перевикористати)

**Що бачить користувач:**
- Ліва панель: конвертований текст з приблизним форматуванням (розмір шрифту візуалізовано)
- Права панель: інструменти анотацій (неактивні, поки не виділено текст)
- Нижня панель: кнопка «Зберегти як шаблон» (неактивна до першої анотації)

**Проблема: втрата форматування з PDF.** PDF не завжди зберігає метадані шрифтів. Якщо інформація відсутня — використати Times New Roman 12pt normal, позначити відповідні параграфи як «форматування невизначене». Користувач може виправити вручну перед збереженням.

**Внутрішнє представлення:**
```json
{
  "paragraph_index": 0,
  "text": "Міністерство освіти і науки України",
  "font": "Times New Roman",
  "font_size": 14,
  "bold": false,
  "italic": false,
  "alignment": "center",
  "space_before_pt": 0,
  "space_after_pt": 6
}
```

### Крок 2 — Розмітка регіонів

Користувач виділяє текст (drag-and-drop у панелі). З'являється спливаюче меню з 4 опціями:

#### Тип A — Зберегти текст і формат

**Лейбл:** «Залишити як є»

Текст і форматування записуються в Python-шаблон як статичний контент. Модель ніколи не змінює цей регіон.

**Використання:** Шапка університету, назва кафедри, логотип.

**Збереження:** `annotation_type: "text_and_format"`

**Згенерований Python:**
```python
p = doc.add_paragraph()
p.alignment = WD_ALIGN_PARAGRAPH.CENTER
run = p.add_run("Міністерство освіти і науки України")
run.font.name = "Times New Roman"
run.font.size = Pt(14)
```

#### Тип B — Зберегти тільки формат

**Лейбл:** «Зберегти формат, текст — модель»

Форматування зберігається, текст стає `{{PLACEHOLDER_KEY}}`. Ключ створюється автоматично.

**Використання:** Заголовки секцій (завжди 16pt bold centered, але текст різний).

**Збереження:** `annotation_type: "format_only"`

**Згенерований Python:**
```python
p = doc.add_paragraph()
p.alignment = WD_ALIGN_PARAGRAPH.CENTER
run = p.add_run("{{SECTION_TITLE}}")
run.font.name = "Times New Roman"
run.font.size = Pt(16)
run.bold = True
```

**Edge case — мультипараграф:** Якщо виділено кілька параграфів з різним форматуванням — створити окремий ключ для кожного. Попередити користувача: «Виділення містить різні стилі форматування. Буде створено N окремих плейсхолдерів.»

#### Тип C — Заміна AI

**Лейбл:** «AI заповнює це»

Регіон стає `{{PLACEHOLDER_KEY}}` без обмежень форматування. Оригінальний текст з файлу додається в інструкції як приклад.

**Використання:** Тіло секції — вступ, методи, результати.

**Збереження:** `annotation_type: "ai_replace"`

**Згенерований Python:**
```python
doc.add_paragraph("{{INTRODUCTION}}")
```

В інструкції автоматично додається:
```
### {{INTRODUCTION}}
AI має згенерувати контент для цього розділу.
В оригінальному документі цей розділ мав текст:
"[оригінальний текст, до 200 символів]..."
Використовуйте як стилістичний reference — не копіюйте.
```

#### Тип D — Іменований розрив (Gap)

**Лейбл:** «Іменований розрив»

Після вибору — діалог із запитом назви (напр. «Footer», «ПІБ студента»). Цей регіон НЕ заповнює AI — користувач вводить вручну при кожному використанні шаблону.

**Використання:** Підпис студента, дата, номер групи.

**Збереження:** `annotation_type: "gap"` з `gap_name: "<назва>"`

**Поведінка при генерації:**
- Gap з'являється як поле вводу у формі (під стандартними полями)
- Текст користувача вставляється в документ без участі AI
- AI може редагувати gap тільки якщо в `user_style` є явна інструкція

**Згенерований Python:**
```python
# GAP: Footer
p = doc.add_paragraph()
p.alignment = WD_ALIGN_PARAGRAPH.LEFT
run = p.add_run("{{GAP_FOOTER}}")
run.font.name = "Times New Roman"
run.font.size = Pt(12)
```

`{{GAP_FOOTER}}` заповнюється з `input_snapshot.gap_values.footer`, не від AI.

### Крок 3 — Огляд і назва

Після всіх анотацій — кнопка «Зберегти як шаблон».

**Діалог:**
- **Назва шаблону** — display name (напр. «Мій формат звіту»)
- **Огляд** — список всіх анотацій з можливістю перейменувати ключі та назви gaps

**Система виконує:**
1. Генерація Python-файлу (див. правила нижче)
2. `ast.parse()` для перевірки синтаксису — якщо помилка, показати і не зберігати
3. Створення `templates` + `custom_template_annotations` в БД
4. **Модальне вікно-попередження:**
   > «Шаблон створено, але інструкції для AI відсутні. Без інструкцій точність заповнення може бути низькою. Додати інструкції зараз?»
   - Кнопка «Додати» → відкрити редактор інструкцій
   - Кнопка «Пропустити» → шаблон доступний, але позначений `has_instructions=0`

### Крок 4 — Інструкції (опційно)

Редактор інструкцій — звичайний текстовий редактор з попередньо заповненим шаблоном:

```markdown
## Призначення шаблону

[Опишіть, для чого цей документ]

## Інструкції для плейсхолдерів

### {{PLACEHOLDER_KEY_1}}
[Що модель має написати тут]

### {{PLACEHOLDER_KEY_2}}
[Що модель має написати тут]

## Правила для зображень

[Які плейсхолдери можуть містити зображення]

## Стиль і тон

[Вимоги до тону, мови, структури]

## Типові помилки

[Що модель має уникати]
```

Ключі плейсхолдерів заповнюються автоматично з `placeholder_schema`.

## Правила генерації Python-файлу

Згенерований файл має бути **сумісним** з рукописними built-in шаблонами (`lab1-template.py`).

1. **Імпорти:**
   ```python
   from docx import Document
   from docx.shared import Pt, Cm
   from docx.enum.text import WD_ALIGN_PARAGRAPH
   from reportlab.lib.pagesizes import A4
   from reportlab.platypus import SimpleDocTemplate, Paragraph, Spacer
   from reportlab.lib.styles import getSampleStyleSheet, ParagraphStyle
   from reportlab.pdfbase import pdfmetrics
   from reportlab.pdfbase.ttfonts import TTFont
   from reportlab.lib.enums import TA_CENTER, TA_JUSTIFY
   from reportlab.lib.units import cm
   ```

2. **Дві функції** (як у built-in):
   - `create_docx(filename)` — створює DOCX
   - `create_pdf(filename)` — створює PDF (через reportlab, **не** libreoffice)

   Текст в обох функціях має бути **ідентичним** (симетрія — необхідно для ManifestValidator).

3. **Анотації в порядку** — ітерувати `annotations`, відсортовані по `paragraph_index`.

4. **`text_and_format`** → `doc.add_paragraph("exact text")` з явним форматуванням.

5. **`format_only`** → те ж форматування, текст = `"{{SECTION_N}}"`.

6. **`ai_replace`** → `doc.add_paragraph("{{PLACEHOLDER_KEY}}")` без форматування.

7. **`gap`** → `doc.add_paragraph("{{GAP_<NAME>}}")` з оригінальним форматуванням.

8. **Збереження:**
   ```python
   doc.save(docx_path)
   # PDF через reportlab — аналогічна логіка
   ```

9. **Заголовок файлу:**
   ```python
   # custom_template_<slug>.py
   # Template: <name>
   # Source: <source_file>
   # Placeholders: KEY_1, KEY_2, GAP_FOOTER
   # Auto-generated by Agent-for-TOM
   ```

10. **Перевірка:** після генерації обов'язково `ast.parse()` для валідації синтаксису.

## Управління інструкціями

Окреме вікно/секція зі списком **всіх інструкцій**:

| Колонка | Значення |
|---|---|
| Назва | `instructions.name` |
| Тип | `global` / `special` / `user_created` |
| Прикріплено до | Назва шаблону або «не прикріплено» |
| Остання зміна | `created_at` |

### Дії
- **Відкрити / Редагувати** — збереження створює нову версію (`is_active=0` для старої)
- **Історія** — всі версії з diff між ними
- **Відкріпити** — `template_id = NULL` (тільки для `user_created`)
- **Прикріпити** — призначити інструкцію шаблону
- **Видалити** — тільки якщо `is_active=0` (стара версія)
- **Створити** — порожній редактор

### Фільтри
- Всі інструкції
- Глобальні
- По шаблону
- Неприкріплені

## Список шаблонів — бейджі

Кожен шаблон у списку показує:
- Назва
- Бейдж: «Вбудований» або «Користувацький»
- Бейдж: «✓ Інструкції» (зелений) або «⚠ Немає інструкцій» (жовтий)
- Посилання «Редагувати інструкції»

**При виборі шаблону без інструкцій:**
> «Для цього шаблону не налаштовано інструкцій AI. Точність контенту може бути неочікуваною. [Додати інструкції →]»

Банер не блокує генерацію — користувач може продовжити.

## Відомі проблеми та рішення

| Проблема | Рішення |
|---|---|
| **Втрата форматування з PDF** | PDF не завжди має метадані шрифтів. Default: Times New Roman 12pt, позначити як «невизначено». Користувач може виправити. |
| **Мультипараграф створює багато ключів** | Попередити. Користувач може об'єднати ключі (той самий ключ для різних параграфів), але попередження про спільний word count budget. |
| **Користувач створив шаблон без інструкцій** | Модальне вікно при створенні + жовтий бейдж у списку. Ніколи не блокувати генерацію. |
| **Інструкції змінилися після старих сесій** | Кожна сесія зберігає `style_hash`. При відкритті старої сесії — інформація: «Інструкції змінилися після створення цієї сесії.» |
| **Gap випадково змінюється AI** | Gap заповнюється з `input_snapshot.gap_values`, AI не бачить ці дані. `editable_by_model=false` за замовчуванням. |
| **Згенерований Python містить помилки** | `ast.parse()` після генерації. Якщо помилка — не зберігати, показати повідомлення. |
